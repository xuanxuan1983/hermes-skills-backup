# GEO（生成式引擎优化）技能文档

基于《中国医美行业GEO可见度报告》（陈萱宜，2026）

## 什么是GEO

GEO = Generative Engine Optimization（生成式引擎优化）

传统SEO优化的是搜索引擎排名，GEO优化的是AI搜索推荐。当用户问ChatGPT、豆包、DeepSeek等AI"哪家医美机构好"时，GEO让你的机构出现在AI的回答中。

## 四维建设法

### 1. 结构化知识库

建立三类核心内容：
- 材料对比库（玻尿酸vs PLLA vs 胶原蛋白）
- 修复方案库（馒化脸、栓塞处理）
- 抗衰方案指南（分年龄段、分预算）

### 2. 权威性信号

| 信号类型 | 具体动作 |
|---------|---------|
| 时间戳 | 内容带发布时间，持续更新 |
| 可验证性 | 数据有来源，观点有论据 |
| 引用关系 | 被权威网站/内容引用 |
| 作者身份 | 明确医生资质+专业背景 |

### 3. 问答场景覆盖

覆盖三类核心问题：
- 选型型："填充材料怎么选"
- 问题型："馒化脸怎么修复"
- 场景型："抗衰项目推荐"

### 4. 跨平台一致性

确保四平台信息一致：
- 机构名称统一
- 核心数据一致
- 定位描述不冲突

## 四平台优化策略速查

### 豆包（权威度优先）

核心策略：
- 展示官方资质（执业许可、医生证书）
- 建立专业背书（论文、行业协会职务）
- 完善Schema标记（MedicalBusiness、Physician）
- 生产攻略型内容（按部位/效果组织）

内容特点：
- 权威来源最多（15条）
- 偏好攻略型回答
- 按部位和效果拆解

### 文心一言（基础信息优先）

核心策略：
- 完善基础信息（名称、地址、电话、营业时间）
- 强化安全字段（风险提示、禁忌人群）
- 使用保守表达（避免绝对化用语）
- 建设百度百科词条

内容特点：
- 回答最保守（0权威来源）
- 偏好要点罗列
- 结构清楚但简短

### DeepSeek（多来源优先）

核心策略：
- 建设多平台内容网络（知乎、小红书、公众号）
- 生产参数化对比内容（维持时间、交联度、价格）
- 建立引用关系（被其他平台引用）
- 使用清单化格式

内容特点：
- 来源最广（35条）
- 偏好清单化、快速比较
- 擅长参数压缩

### 千问（深度内容优先）

核心策略：
- 生产深度长文（2000字以上）
- 提供完整方案（梯队推荐+避坑提醒）
- 场景化内容（预算规划+时间规划）
- 建立FAQ体系

内容特点：
- 回答最详尽（2156字）
- 完整决策建议
- 包含梯队推荐

## 技术工具

### 1. llms.txt生成器

位置：~/.claude/skills/medaesthetic-geo-suite/scripts/llmstxt_generator.py

功能：
- 生成llms.txt（标准版，针对豆包/DeepSeek）
- 生成llms-full.txt（完整版，针对千问，2000+字）

使用：
```
python3 llmstxt_generator.py clinic-config.json ./output
```

### 2. Schema生成器

位置：~/.claude/skills/medaesthetic-geo-suite/scripts/schema_generator.py

功能：
- 生成MedicalBusiness Schema（豆包优化）
- 生成Physician Schema（豆包优化）
- 生成Product Schema（DeepSeek优化，含高频材料数据）

使用：
```
python3 schema_generator.py complete clinic-config.json
```

### 3. 报告生成器

位置：~/.claude/skills/medaesthetic-geo-suite/scripts/geo_report_generator.py

功能：
- 生成Markdown报告
- 生成PDF报告（需要reportlab）

使用：
```
python3 geo_report_generator.py audit-data.json report.md
python3 geo_report_generator.py audit-data.json report.pdf
```

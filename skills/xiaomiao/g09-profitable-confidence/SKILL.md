# 术语: 盈利自信 (Profitable Confidence)

## R — 原文 (Reading)

> 盈利能力是一种超能力...盈利能力让你脱离电网，在无限的跑道上可以有意识地发展企业。

> — 萨希尔·拉文吉亚, 第六章

---

## I — 方法论骨架 (Interpretation)

盈利自信不只是一个会计概念——它是一种**结构性能力：停止主动销售后公司还能存活多久**。

核心逻辑:
1. 传统"盈利" = 有净利润（绝对值）
2. "盈利自信" = 产品力带来的结构性留存（非创始人努力）

关键测试:
- 关闭所有营销活动，看收入下降的速度
- 每月下降<5% = 盈利结构健康
- 每月下降50% = 收入依赖你的时间投入

盈利给了你"可以说不"的能力——对投资人说不，对糟糕的交易说不。

## A1 — 书中的应用 (Past Application)

### 案例 1: Gumroad股权众筹
- **问题**: 为什么盈利后才能做众筹而不是VC
- **方法论的使用**: 盈利后才有底气做众筹
- **结论**: "盈利能力让你脱离电网，在无限的跑道上可以有意识地发展企业"
- **结果**: 成功完成众筹

### 案例 2: 从追逐独角兽到收缩
- **问题**: 为什么追逐规模反而失去自信
- **方法论的使用**: 作者亲身经历了两种状态
- **结论**: 追逐独角兽需要投资人，自信需要盈利
- **结果**: 收缩后实现盈利，获得自信

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户在讨论盈利的意义——不只是"赚钱"，而是"自由"
2. 用户说"增长比盈利重要"——需要重新定义优先级
3. 用户问"什么时候该追求盈利"——极简主义的时机
4. 用户说"盈利是目标"——需要理解什么是真正的盈利
5. 用户在问"要不要接受这笔投资"——盈利是谈判筹码

### 语言信号 (用户的话里出现这些就应激活)

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **测试"关闭营销收入下降多少"**
   - 完成标准: 用户关闭营销活动一周，观察收入下降百分比

2. **判断盈利结构健康度**
   - 完成标准: 每月下降<5% = 健康；每月下降>30% = 结构性问题

3. **制定"盈利自信"路线图**
   - 完成标准: 用户设定"什么收入水平可以实现盈利自信"的具体目标
   - 判停条件: 若用户已有盈利自信，可讨论扩张；否则继续聚焦Default Alive

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 需要大量资本投入才能启动的行业——那些行业确实需要融资
- 讨论上市前的财务规划——那是另一套逻辑
- 把"盈利"当成不做事的借口——盈利是手段，不是目的

### 作者在书中警告的失败模式

- 把"有净利润"当成"盈利自信"——结构性问题被掩盖
- 依赖创始人销售而非产品力——创始人离开，收入归零
- 为了盈利牺牲增长——需要在两者之间找到平衡

### 容易混淆的邻近方法论

- "现金流为王" — 那是短期生存概念，本方法是长期自信概念
- "利润率最大化" — 那是财务指标，本方法是战略能力

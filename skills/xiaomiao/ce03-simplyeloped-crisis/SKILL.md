# 反例: Simply Eloped的文化危机与重建

## R — 原文 (Reading)

> 我们招了朋友和家人，还有任何看上去很友好并想要一份工作的人。结果是：欺凌、流言蜚语、冲突不断。

> — 萨希尔·拉文吉亚, 第七章

---

## I — 方法论骨架 (Interpretation)

这个反例揭示了**早期文化匹配比技能匹配更重要的原则**。

核心逻辑链:
1. 基于关系招聘（朋友、家人）→ 价值观不匹配
2. 价值观不匹配 → 内部冲突、自我强化
3. 内部冲突 → 文化危机、效率下降
4. 文化危机 → 需要外部干预（领导力教练）
5. 干预成本 >> 预防成本

反直觉教训: 技能可以通过学习提升，但价值观冲突是结构性的，不会随时间消失。

---

## A1 — 书中的应用 (Past Application)

### 案例 1: Simply Eloped的文化危机
- **问题**: 招聘基于关系而非价值观，导致欺凌、流言蜚语
- **方法论的使用**: 在建立文化之前就开始招聘
- **结论**: 文化问题一旦积累，解决成本极高
- **结果**: 请了领导力教练才重建

### 案例 2: Simply Eloped的重建
- **问题**: 如何从文化危机中恢复
- **方法论的使用**: 重新定义价值观，并让不匹配的人离开
- **结论**: 文化重建需要时间和外部帮助
- **结果**: 重建成功但成本巨大

---

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户说"这是朋友，应该没问题"——用关系代替价值观匹配
2. 用户说"我们需要人手，先招再说"——紧迫性压倒文化考量
3. 用户招聘时感觉候选人"不太好沟通"——已有信号但不确定是否重要
4. 用户问"文化重要还是技能重要"——需要明确优先级
5. 用户说"先招进来，不合适再换"——低估文化问题的粘性

### 语言信号 (用户的话里出现这些就应激活)

- "这是朋友，应该没问题"
- "我们需要人手，先招再说"
- "技术好就行"
- "面试感觉不太对但又说不上来"
- "先试试看"

### 与相邻 skill 的区分

- 与 `p10-合适是双向的` 的区别: 本skill是"为什么文化匹配优先",p10是"如何判断双向匹配"
- 与 `p17-价值观早定常定` 的区别: 本skill强调招聘前的文化重要性,p17强调文化持续性

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **问用户"愿意和他一起解决冲突2年吗"**
   - 完成标准: 用户回答了"是"或"否"，并能说出原因

2. **识别文化问题信号**
   - 完成标准: 用户能列出面试中观察到的不匹配信号（沟通方式、决策风格等）

3. **权衡成本**
   - 完成标准: 用户理解"招聘错误的成本是事先不可见的"，而文化危机的成本是事后可见的
   - 判停条件: 若用户说"不匹配的成本超过2年经验的收益"，则建议不录用

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 已有清晰文化定义的成熟公司——他们有机制处理文化不匹配
- 非关键岗位的临时招聘——短期实习/外包不需要深度文化匹配
- 用户已明确知道文化且有筛选标准——跳过此skill

### 作者在书中警告的失败模式

- 在建立文化之前就开始招聘
- 招聘基于关系而非价值观匹配
- 忽视早期文化问题的信号

### 如何避免 Simply Eloped 式的失败

1. **先定义文化，再招聘** — 在招第一个员工之前，写下3-5条"我们相信"
2. **问"2年测试"** — 问自己："我愿意在接下来2年里每周和他一起解决冲突吗？"如果答案是否，价值观不匹配的成本会超过技能收益
3. **不要用"朋友"代替"适合"** — 如果你已觉得某人不对，他大概率也觉得你不对
4. **重视面试中的"不太好沟通"信号** — 不要期待入职后会改善，这是结构性而非偶然性问题

### 容易混淆的邻近方法论

- "技能优先"招聘——本skill不否定技能，而是说技能可学但价值观冲突不可解
- "文化契合"（通常指同质化）——本skill强调互补性而非同质性

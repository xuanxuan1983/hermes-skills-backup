# 合适是双向的原则

## R — 原文 (Reading)

> "当某人对你不合适，意味着你对他也不合适。"

— Sahil Lavignena, 第七章

---

## I — 方法论骨架 (Interpretation)

这个原则的核心洞察是：招聘不是单向评估，而是双向选择。当你观察一个候选人"是否合适"时，你也应该意识到：对方同时在评估"这个公司是否适合我"。

如果一个人让你觉得"不合适"，你观察到的只是单向信号。真正的双向评估会问：他是否也觉得这个团队不适合他？

如果答案是"是"——他也不满意，那么继续合作对双方都是损失，应该尽早分手。

如果答案是"否"——他很满意，那么问题在于你的期望是否合理，需要你自己调整。

这个原则的推论是：不要强留任何人。强留只会让你花更多时间管理他，而他也不会快乐。坦诚沟通、帮助离职、提供推荐信，是更好的选择。

**核心洞察：合适是相互的评估，不是单向的审判。双向不匹配应尽早分手。**

---

## A1 — 书中的应用 (Past Application)

### 案例 1: Simply Eloped 的文化危机（反例）

- **问题**: Simply Eloped 早期招了"朋友和家人，还有任何看上去很友好并想要一份工作的人"
- **方法论的使用**: 没有用"双向合适"原则筛选，而是用关系和友好度判断
- **结论**: 导致内部冲突、价值观混乱
- **结果**: 后来通过 CACAO 框架重建，明确价值观后让不匹配的人自然离开

### 案例 2: Simply Eloped 的重建

- **问题**: 文化危机后需要重建团队
- **方法论的使用**: 定义明确的 CACAO 价值观（沟通、成就、手艺、自主、所有权）
- **结论**: 用价值观过滤，留下真正匹配的人
- **结果**: 危机后重建了健康的团队文化

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill？

1. 用户在面试候选人，在问"这个人合不合适"
2. 用户有一个表现一般的员工，在考虑"要不要给他机会改进"
3. 用户说"他技术很强，但沟通方式让人不舒服"
4. 用户在困惑"我怎么知道候选人是否真的想来"
5. 用户在招聘早期员工，想知道"怎么找到志同道合的人"

### 语言信号 (用户的话里出现这些就应激活)

- "这个人合不合适"
- "要不要开除他"
- "价值观不匹配"
- "他技术很强但"
- "我想给他机会改进"
- "怎么知道候选人是否真的想来"

### 与相邻 skill 的区分

- 与 `p17-values-early-often` 的区别: p17 关注"价值观什么时候建立"，p10 关注"怎么用价值观评估人"；p17 是建立，p10 是应用
- 与 `p09-transparency` 的区别: p09 关注"日常运营中的信息对称"，p10 关注"招聘和裁员中的决策"

## E — 可执行步骤 (Execution)

当 skill 被激活后， agent 应按以下步骤执行:

1. **诚实地问自己：谁让你不舒服？**
   - 完成标准: 列出你团队中让你有负面反应的人（不只是绩效低的）
   - 区分：是因为能力不足（可以培养），还是因为价值观不匹配（难以改变）？

2. **问：他是否也觉得你不合适？**
   - 完成标准: 观察他的行为——他是否主动承担额外责任？他是否在团队中表现积极？
   - 如果他很满意，你需要决定：这个不适感是否值得你去改变

3. **如果是双向不匹配，尽早分手**
   - 完成标准: 不要等到问题恶化；开始坦诚对话
   - 提供：帮助离职、推荐信，而不是"给他一个最后机会"

4. **如果是单向不匹配（他不满意，你满意），帮他找到更匹配的环境**
   - 完成标准: 承认公司可能不适合他
   - 问：你是否能改变环境来留住他？如果不能，帮助他离开

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 你还没有定义明确的价值观——双向合适需要标准才能判断
- 你的问题只是沟通不畅——这可以通过 coaching 解决，不是价值观问题
- 你处于极度需要人手的情况下——等你有资源时再执行这个原则
- 员工正在经历个人危机（家庭问题、健康问题）——这不是文化问题

### 作者在书中警告的失败模式

- "如果你在想'我要不要开除他'，答案通常是要"
- "给他机会改进通常是给自己挖坑——如果他不匹配，他很难改变"

### 作者的盲点 / 时代局限

- 早期创业公司可能需要"不那么合适"但有能力的人——在资源有限时，能力可能比文化匹配更重要
- "尽早分手"在某些文化中是困难的——可能需要过渡期或补偿
- 对于远程团队，评估"文化匹配"可能更难

### 容易混淆的邻近方法论

- 价值观招聘: 价值观招聘是"用价值观筛选候选人"，p10 是"用价值观评估现有员工"；两者都是 p17 的应用
- 人才评估矩阵: 矩阵是"评估能力/潜力"，p10 是"评估双向合适性"；矩阵是工具，p10 是判断

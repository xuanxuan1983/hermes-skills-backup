# 透明度原则

## R — 原文 (Reading)

> "我们的目标是信息完全对称。我所知道的信息，你们也都知道，也不会有你和我知道但我们的创作者不知道的信息。"

— Sahil Lavignena, 第七章

---

## I — 方法论骨架 (Interpretation)

这个原则的核心是消除组织内的信息差。Sahil 认为：信息不对称是管理问题的根源——当创始人知道而员工不知道时，员工会做出错误的决策；当员工知道而创始人不知道时，创始人会错过重要信号。

透明度的具体实践包括：
- **财务数据公开**: 所有员工都能看到收入、成本、融资
- **薪资数据公开**: 没有暗箱操作的薪酬谈判
- **决策过程透明**: 不是只有结果公开，而是决策的依据和过程都公开

透明度的价值不在于"民主决策"，而在于**减少决策失真**。当每个人都有同样的信息时，决策质量会更高，不需要反复解释"为什么这么做"。

**核心洞察：透明度不是为了民主，而是为了消除信息差导致的决策失真。**

## A1 — 书中的应用 (Past Application)

### 案例 1: Gumroad 的薪资公开

- **问题**: 传统公司里薪资不透明导致内部不公平感和猜疑
- **方法论的使用**: Gumroad 公开所有员工的薪资，包括创始人
- **结论**: 任何人都能查到任何人的工资
- **结果**: 减少了内部政治，让薪酬讨论变得透明

### 案例 2: WeWork 的不透明危机（反例）

- **问题**: WeWork CEO 独自决策1300万美元投资冲浪公司，没有向董事会披露
- **方法论的使用**: 信息不透明导致失控的权力
- **结论**: 缺乏透明度是治理失败的根源
- **结果**: 公司最终崩溃，估值从470亿跌到80亿

## E — 可执行步骤 (Execution)

当 skill 被激活后， agent 应按以下步骤执行:

1. **建立信息记录系统**
   - 完成标准: 创始人每周写一段"状态更新"（100-200字），记录本周决策、进展、困惑
   - 分发：发给所有团队成员，无论大小

2. **公开财务数据**
   - 完成标准: 每月向全团队展示关键财务指标（收入、成本、利润/亏损）
   - 用简单语言解释：不是专业财报，而是"我们现在的情况"

3. **公开决策过程**
   - 完成标准: 重大决策前，公开决策背景和考量因素；决策后，解释结果
   - 问自己：如果团队成员能看完整邮件链，他们会理解这个决定吗？

4. **建立反馈闭环**
   - 完成标准: 定期（每月）收集团队成员对公司信息透明度的反馈
   - 问：他们是否觉得信息够了？他们最困惑的是什么？

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 你正在融资或被并购——某些财务信息需要保密
- 你只有你自己——透明度是团队属性，一个人不需要向自己透明
- 你的团队成员都是短期合同工——他们可能不需要深度业务信息
- 涉及员工个人隐私的信息（医疗、绩效警告）——这类信息不应该全公司公开

### 作者在书中警告的失败模式

- "透明度不是民主。你仍然做最终决定，但你要让所有人知道为什么"
- "如果你发现自己在隐藏信息，问问自己：为什么？如果理由是'保护他们'，你可能高估了自己的判断"

### 作者的盲点 / 时代局限

- 过度透明可能导致决策瘫痪——当所有人都能看到所有信息时，可能过度参与每个决定
- 在某些文化背景下（如中国），完全透明可能引起恐慌而非信任
- 信息过载可能成为负担——不是所有信息都需要平等地传递给所有人

### 容易混淆的邻近方法论

- 扁平化管理: 扁平化是"减少层级"，透明度是"减少信息差"；两者相关但不同
- OKR 体系: OKR 是"目标对齐工具"，透明度是"信息对称机制"；OKR 是透明度的一种实现方式

# 术语: 极简主义创业者 (Minimalist Entrepreneur)

## R — 原文 (Reading)

> 极简主义创业者...关注如何"尽一切力量盈利"，而不是"不计一切成本扩张"。

> — 萨希尔·拉文吉亚, 第一章

---

## I — 方法论骨架 (Interpretation)

极简主义创业者是一种**主动选择"足够"而非被动"做不大"的创业策略**。

核心特征:
1. 盈利优先于规模 — 先止血，再谈增长
2. 用最少资源验证 — 不融资也能启动
3. 有意识地控制增长 — 什么时候扩张，我说了算
4. 社区驱动而非融资驱动 — 先有客户，再有钱

这不是"小本生意"（被动保守），而是一种**主动选择何时、如何增长**的自由哲学。

## A1 — 书中的应用 (Past Application)

### 案例 1: Gumroad的演变
- **问题**: 从追逐独角兽到接受极简主义
- **方法论的使用**: 作者亲身验证两种状态
- **结论**: "极简主义创业者...关注如何'尽一切力量盈利'"
- **结果**: 收缩后实现盈利，获得自主权

### 案例 2: 贯穿全书的反面案例
- **问题**: Quibi、WeWork追逐规模的失败
- **方法论的使用**: 用反例验证极简主义的正确性
- **结论**: 追逐规模不等于成功
- **结果**: 反面案例与正面案例共同构建极简主义框架

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 用户问"什么是极简主义创业"——需要理解核心定义
2. 用户在对比MBA思维与传统创业——需要区分极简主义与这两种
3. 用户说"小公司如何生存"——把极简主义当生存策略
4. 用户问"融资是不是必须的"——重新定义融资的角色
5. 用户说"我想小而美但不知道怎么开始"——需要可执行路径

### 语言信号 (用户的话里出现这些就应激活)

- "小而美"
- "保守经营"
- "生活方式创业"
- "用最少资源"
- "先盈利再扩张"

### 与相邻 skill 的区分

- 与 "传统创业者" 的区别: 传统追求规模、融资、退出；极简追求盈利、自主、有意识增长
- 与 "生活方式创业" 的区别: 极简不排斥增长，只是选择何时增长
- 与 "精益创业(Eric Ries)" 的区别: 精益追求快速迭代和增长；极简追求有意识地控制增长

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **重新定义成功指标**
   - 完成标准: 用户把"盈利"设为首要指标，而非"用户数"、"估值"

2. **从"融资"移到"盈利之后"**
   - 完成标准: 用户理解"先实现Default Alive，再考虑融资"

3. **找到你的"足够"**
   - 完成标准: 用户明确"足够"是什么（多少收入？多少客户？），而不是无止境追求更大
   - 判停条件: 若用户已有清晰的"足够"定义，极简主义可以开始实践

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- 需要大量资本的硬件/生物科技创业——那些领域确实需要融资
- 讨论上市战略——那是另一套逻辑
- 把极简主义当成"不努力"的借口——极简是选择，不是懒惰

### 作者在书中警告的失败模式

- 把极简主义当成保守的借口
- 不追求增长，只追求"活着"
- 用"小而美"逃避市场竞争

### 容易混淆的邻近方法论

- "小本生意" — 极简是主动选择，小本是被动状态
- "生活方式创业" — 极简不排斥增长，只是选择何时增长

# 萱宜公众号排版模板 v29（定稿）

## 核心结构规则

1. **标题不重复**：WeChat 自动从 --title 参数显示标题，HTML body 里**不要写 h1**，否则标题显示两遍
2. **顶部元信息**：文章最上方只放 `共 XXXX 字 · 阅读约 N 分钟`
3. **作者区无框无横线**：不要 background/border/padding 嵌套，直接：圆形头像 + 名字 + 简介段落

## 封面图生成（Hardcore Retro-Tech Gem 2.0）

用户给文章主题 → 我填 JSON 生成 prompt → MiniMax 图像 API → 保存到桌面

```json
{
  "Gem_System": "Hardcore Retro-Tech Gem 2.0 (Full Auto)",
  "Article_Content": {
    "Title": "[文章标题]",
    "Keywords": "[3个核心英文关键词]",
    "Scene_Description": "[一个具体物理动作场景，英文]"
  },
  "Style_Preferences": {
    "Vibe": "educational manual print, Riso print",
    "Complexity": "wide angle, focused on detail"
  }
}
```

配色自动从4对互补色随机抽取：铜+钴蓝/橙+青绿/绿+品红/黄+紫。

生成命令：
```bash
MINIMAX_API_KEY="sk-api-..." MINIMAX_BASE_URL="https://api.minimaxi.com/v1" \
  bun scripts/main.ts --prompt "In the style of image_0.png, ..."
```

## 迭代中踩过的坑

- **作者照片重复**：文章顶部写真图不要在作者区再放一张
- **v22 vs v29**：极简留白 > 堆砌装饰；happycapy 风格编号+大量留白是对的
- **回退记录**：v21太素、v26有重复标题、v27引言重复 → 以 v29 为基准
